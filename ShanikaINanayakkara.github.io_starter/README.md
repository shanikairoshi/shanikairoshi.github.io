## Important GitHub Pages naming rule

To publish at:

```text
https://shanikairoshi.github.io/
```

the GitHub account or organisation owner must be named:

```text
ShanikaINanayakkara
```

and the repository must be named:

```text
ShanikaINanayakkara.github.io
```

If your current GitHub username remains `shanikairoshi`, use one of these options:

1. rename your GitHub username to `ShanikaINanayakkara`;
2. create a GitHub organisation called `ShanikaINanayakkara`;
3. keep the URL `https://shanikairoshi.github.io/` but use the site title **Shanika I. Nanayakkara**;
4. buy a custom domain later and point it to GitHub Pages.

## How to publish

1. Create a public GitHub repository named `ShanikaINanayakkara.github.io`.
2. Upload all files from this folder into the repository root.
3. Go to **Settings → Pages**.
4. Under **Build and deployment**, choose:
   - Source: `Deploy from a branch`
   - Branch: `main`
   - Folder: `/root`
5. Save.

Search all files for `[Add` and `[ADD]` and replace placeholders.

## Do not publish

- IBM Quantum API tokens
- private dataset files
- confidential supervisor comments
- unpublished reviewer comments
- large raw CSV files
- `.env` files
- virtual environments
- private thesis examination documents
